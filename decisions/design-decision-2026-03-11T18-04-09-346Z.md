# Design Decision: UI Background Color Heuristic for Tooltips and Menus

**Author:** Cary Snyder
**Status:** Accepted
**Date proposed:** March 10, 2026
**Date approved:**

## Problem

The team encountered a white-on-white visibility issue when implementing chart tooltips for synthetic readings, particularly problematic when displaying white-colored data series like rate charts. This prompted a broader discussion about when to use white vs. dark backgrounds for different UI components. The engineering team was blocked on ticket APP-191 and needed clear guidance on tooltip styling to maintain visual consistency across the application.

## Decision

Establish a UI background color heuristic based on component function:
- **White backgrounds** = High-level, interruptive UI (Dropdowns, menus)
- **Dark backgrounds** = Contextual/supplementary data (Chart tooltips, trend tooltips)

Chart tooltips will use dark backgrounds with appropriate styling to match the designed specifications, including moving color indicators to the right side of the tooltip layout.

## Consequences

### Positive
- Resolves white-on-white visibility issues for chart tooltips
- Creates clear, functional distinction between different UI component types
- Reduces visual distraction when viewing charts and data tables
- Provides consistent styling guidance for future development
- Unblocks engineering team immediately
- Dark tooltips provide a "sexier" marketable appearance for buyer personas

### Negative / Tradeoffs
- Dark tooltips may get lost in visual clutter among other grey elements on the page
- Motion/interaction may be required to provide sufficient contrast for some users
- Decision is explicitly temporal and may need revisiting for light mode implementation

### Neutral
- Mobile experience considerations noted but not fully explored
- Light mode compatibility deferred for future evaluation

## Alternatives Considered

| Alternative | Why Not Chosen |
|---|---|
| High contrast white tooltips for all components | Too visually distracting, especially problematic on mobile where large tooltips would dominate the screen |
| Adding black borders around white elements in white tooltips | More complex implementation; doesn't address the broader consistency issue |
| Context-dependent styling without clear heuristic | Would require case-by-case decisions and lack systematic consistency |

## Additional Context

This decision is explicitly noted as temporal and valid based on current constraints in dark mode. The team acknowledges this heuristic will likely need revisiting when implementing light mode for the Titan product. Engineering had already implemented working versions of both approaches, making the transition straightforward once the decision was finalized.
